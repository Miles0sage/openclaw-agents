How It Works — Interactive Walkthrough

1
Task Input ▼

You give the system a task: "Build a dashboard for my AI agents" or "Find security vulnerabilities in my code."

The task is received by the Overseer, who reads it carefully and decides whether to handle it directly or delegate to a specialist.

2
Smart Routing ▼

The Overseer looks at the task type and matches it to the cheapest capable agent:

Simple code task → CodeGen Pro ($0.14/1M)

Security audit → Pentest AI ($0.27/1M)

Database query → SupabaseConnector ($15/1M, but necessary)

This saves you money — 100x cheaper than using Opus for everything.

3
Agent Execution ▼

The chosen agent has access to its specialized tools:

CodeGen Pro's tools: file_edit, git_operations, auto_test, shell_execute

Researcher's tools: web_search, deep_research, save_memory

Code Reviewer's tools: file_read, git_operations, codex_query

Each agent uses its tools to solve the problem. Tools are restricted per agent for safety.

4
Ralph Gate ▼

After the agent finishes, a quality gate checks the output:

✓ Does the code compile?

✓ Do the tests pass?

✓ Is the confidence score above the threshold?

If confidence is low, the system retries with a more capable (expensive) model.

5
Code Review ▼

The Code Reviewer agent votes on quality:

🔍 Logic errors? Are edge cases handled?

🔍 Architecture? Does it fit the codebase?

🔍 Security? Any vulnerabilities?

If 2 of 3 reviewers approve, the output is good to ship.

6
Delivery ▼

The completed task is delivered back to you:

✓ If it's code: committed to git and pushed

✓ If it's research: a structured report with citations

✓ If it's analysis: formatted data with visualizations

Each result includes cost and duration metrics.

Learning — What Is This System?

What is an AI Agent?

An AI agent is a program that can think and act independently. Unlike a chatbot that you ask questions to, an agent:

✓ Breaks goals into steps — "Build a dashboard" becomes 10 subtasks

✓ Chooses tools to use — reads files, runs tests, makes git commits

✓ Recovers from errors — if a test fails, it debugs and retries

✓ Knows what it doesn't know — escalates to a more capable agent when needed

What is MCP (Model Context Protocol)?

MCP is the "nervous system" connecting AI agents to tools. It's a standard way for agents to ask for information:

Agent asks: "Read this file for me"

MCP responds: "Here's the file content"

Instead of hardcoding every tool into the agent, MCP lets agents discover and use tools dynamically. OpenClaw uses 75+ MCP tools across security, database, web, and code domains.

What is a Tool?

A tool is a function that solves one problem well. Examples:

file_read — Read a file safely

git_operations — Commit and push code

auto_test — Run tests and report failures

web_search — Search the internet for current info

security_scan — Find vulnerabilities

Agents combine tools like building blocks to accomplish complex tasks.

How Do Agents Coordinate?

OpenClaw uses a simple protocol:

1. Overseer receives task from you

2. Overseer routes to the best specialist

3. Specialist works and reports back

4. Overseer verifies before responding to you

This prevents chaos — specialists stay in their lane, and the Overseer keeps everything on track.

Why Multiple Models?

Cost vs. Quality. Every model makes a tradeoff:

Kimi 2.5: $0.14/1M — Fast, cheap, good for bounded tasks

MiniMax M2.5: $0.30/1M — Better reasoning, good for architecture

Claude Opus: $15/1M — Best reasoning, use when accuracy matters

OpenClaw uses the cheapest model that works, and escalates if needed. This saves 100x on costs while maintaining quality.

What Happens If Something Breaks?

The system recovers automatically:

First failure: Diagnose the error and retry with same model

Second failure: Escalate to a more capable model

Third failure: Mark as permanent, report to Overseer

This "failure recovery protocol" means you rarely see stuck tasks. The system self-heals.
